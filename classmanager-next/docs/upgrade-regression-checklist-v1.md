# 升级改造阶段回归清单 v1

## 1. 目的

本文档用于把当前升级改造阶段已经完成的高频模块收成一份可执行回归清单，避免后续验收时只靠记忆。

阶段整体状态可配合 `docs/upgrade-stage-status-v1.md` 一起查看。
人工走查结果建议记录到 `docs/upgrade-acceptance-record-template-v1.md`。

当前覆盖范围：

- 学生档案
- 积分手工调整
- 考勤写操作
- 作业写操作
- 设置写操作
- 管理后台
- 导出任务中心

## 2. 自动验证基线

每次准备合并或阶段交付前，至少执行以下命令：

```bash
npm run build -w @classmanager/web
npm test -w @classmanager/api -- src/modules/points/routes.test.ts
npm test -w @classmanager/api -- src/modules/attendance/routes.test.ts
npm test -w @classmanager/api -- src/modules/homework/routes.test.ts
npm test -w @classmanager/api -- src/modules/settings/routes.test.ts
npm test -w @classmanager/api -- src/modules/admin/routes.test.ts
npm test -w @classmanager/api -- src/modules/exports/routes.test.ts
```

说明：

- 当前 `@classmanager/api` 的测试脚本会把 `src/**/*.test.ts` 全量执行，因此上面任一 `npm test -w @classmanager/api -- ...` 本质上都会覆盖整套 API 测试。
- 前端改动后至少跑一次 `@classmanager/web` 构建，确认类型和 JSX 没有回归。

## 3. 人工回归清单

### 3.1 学生档案

- 学生列表可正常按姓名、小组、宿舍筛选。
- 学生列表批量组织归属、岗位、状态调整入口仍可正常打开。
- 学生详情基础资料可编辑并保存。
- 学生详情组织归属区能看到“当前值 / 保存后”预览。
- 学生详情最近积分记录可按类型、撤销状态、关键词筛选。
- 学生详情最近积分记录摘要会随筛选变化。

### 3.2 积分手工调整

- 学生列表最近积分操作可按类型、撤销状态、日期范围、关键词筛选。
- 最近积分操作摘要会正确显示单条/批量/可撤销数量。
- 历史批量调整可按类型、日期范围、理由/场景/类别/人数/分值/批次搜索。
- 历史批量调整摘要会正确显示加分批次、扣分批次、影响学生。
- 单条撤销、最近批量撤销、按 `batchId` 撤销历史批次入口都仍可见。

### 3.3 考勤写操作

- 考勤时段配置可正常新增、调整顺序、停用并保存。
- 删除带历史数据的考勤时段后，会以“停用保留历史”的方式回显，不会误删历史场次。
- 历史场次可按日期、时段、场次状态、关键词筛选。
- 场次明细可按状态、学生姓名/学号/备注筛选。
- 场次明细汇总会随筛选变化。
- 异常工作台可按异常类型、关键词筛选，并能正确显示待结算缺勤数量。
- 异常工作台支持跨场次批量修正与缺勤结算。
- 历史批量操作可按类型、状态、人数、日期、批次搜索，并支持排序。
- 最近操作可按类型、撤销状态、日期范围、关键词筛选。
- 最近操作摘要会正确显示单条修正、批量修正、批量补录、可撤销数量。

### 3.4 作业写操作

- 最近作业操作可按类型、撤销状态、日期范围、关键词筛选。
- 作业明细可按事件类型、学生关键词筛选，并支持排序。
- 学生作业统计可按需关注/正常、学生关键词筛选，并支持排序。
- 历史批量作业登记可按类型、人数、日期、批次搜索，并支持排序。
- 单条撤销、审计撤销、按 `batchId` 撤销历史批次入口仍可正常显示。

### 3.5 设置写操作

- 积分模板可按类型、状态、关键词筛选，摘要计数正确。
- 功能开关可按状态、关键词筛选。
- 倒计时事件可按标题、备注、日期搜索。
- 课程备注可按键名、内容搜索。
- 倒计时与课程备注筛选后计数和空状态显示正常。

### 3.6 管理后台

- 成员列表可按搜索、状态、角色、排序、操作视角联合筛选。
- 成员列表筛选摘要能正确显示正常/停用/待激活/可删除数量。
- 成员详情操作摘要卡片能正确反映当前限制。
- 角色分配区能看到“当前角色 / 保存后角色”预览。
- 邀请表单能看到 `invited` 状态与角色预览。
- 成员审计与后台审计可按动作、关键词筛选。
- 后台审计摘要能正确显示动作分布。

### 3.7 导出任务中心

- 最近任务可按导出域、状态、归属、关键词筛选。
- 最近任务摘要能正确显示成功、失败、排队/执行中、过期、可下载数量。
- 导出清单可按来源、导出域、关键词筛选。
- 导出审计可按动作、关键词筛选。
- 下载按钮只在允许下载的任务上展示。

## 4. 阶段性通过标准

满足以下条件，可视为当前升级阶段通过：

1. 前端构建通过。
2. API 测试全量通过。
3. 上述人工回归清单中高频入口均可正常进入。
4. 最近新增的筛选、摘要、预览不影响原有写入和撤销入口。
5. 文档与实现状态一致，不再保留明显过时描述。

## 5. 当前剩余风险

- 多数新增治理能力属于前端本地筛选和摘要，仍需要一次真实数据库人工走查确认展示字段与真实业务习惯一致。
- 部分 `npm test -w @classmanager/api -- ...` 命令虽然能覆盖全量测试，但脚本行为和命令字面含义不一致，后续可单独整理测试入口。
- 写操作主链路虽然已具备测试和多轮构建验证，但跨模块联合回归仍建议至少做一次真实班级演练。
