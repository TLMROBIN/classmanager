## 2. 优先级建议

### P0 验收收口

- 按 `docs/upgrade-regression-checklist-v1.md` 做一次真实人工走查。
- 将结果补入 `docs/upgrade-acceptance-record-initial-v1.md`。
- 对人工走查中发现的问题做最后一轮修正。

### P1 核心写操作补尾

- 积分：
  - 更复杂批量写入能力评估
  - 批量修正 / 撤销真实回归与边界治理
  - 更明确的撤销影响提示继续收口
- 考勤：
  - 更细规则项
  - `excused` / 缺勤结算 / 异常工作台真实回归评估
  - 跨动作统一撤销策略评估
- 作业：
  - 批量修正流真实回归与边界治理
  - 独立作业记录模型是否需要启动
- 学生档案：
  - 更完整的学生状态治理
  - 归档/删除策略设计

### P2 后台与导出治理

- 管理后台真实数据库回归
- 更细粒度租户治理权限
- 导出任务中心真实下载链路走查
- 导出文件治理与独立 worker 方案评估

### P3 样本为空模块

- 任务系统
- 藏宝阁 / 商城
- 对战系统

前提：

- 先拿到真实旧数据样本
- 再决定是否建模与迁移

## 3. 建议执行顺序

1. 先做 P0，完成人工验收与问题收口。
2. 再做 P1，补齐核心写操作剩余缺口。
3. 再做 P2，推进后台治理和导出治理进入真实运维状态。
4. 最后再考虑 P3。
